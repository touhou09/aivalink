# AivaLink 유저 플로우 문서

## 1. 주요 사용자 시나리오 개요

| # | 시나리오 | 관련 기능 | Phase |
|---|----------|-----------|-------|
| 1 | 첫 가입 → 캐릭터 설정 → VTuber 시작 | 인증, 캐릭터, 설정, 인스턴스 | 1~3 |
| 2 | LLM 제공자 변경 (API 키 vs OAuth) | LLM 설정 | 1, 3 |
| 3 | TTS 커스텀 음성 모델 업로드 및 적용 | TTS 설정, 파일 업로드 | 1, 3 |
| 4 | 실시간 대화 (음성/텍스트) | WebSocket, ASR, LLM, TTS, Live2D | 2, 3 |
| 5 | 에이전트 도구 사용 (MCP 로컬 PC 접근) | 에이전트, MCP | 4 |
| 6 | 캐릭터 전환 | 인스턴스 관리, 캐릭터 | 2, 3 |

---

## 2. 시나리오 1: 첫 가입 → 캐릭터 설정 → VTuber 시작

### 2.1 전체 흐름

```
+--------+     +------------+     +-------------+     +------------+     +-----------+
| 가입/  | --> | LLM 설정   | --> | TTS 설정    | --> | ASR 설정   | --> | 캐릭터    |
| 로그인 |     | 생성       |     | 생성        |     | 생성       |     | 생성      |
+--------+     +------------+     +-------------+     +------------+     +-----+-----+
                                                                               |
                                                                               v
                                                                        +-----------+
                                                                        | VTuber    |
                                                                        | 시작!     |
                                                                        +-----------+
```

### 2.2 단계별 상세

#### 단계 1: 회원가입

```
사용자                              프론트엔드                           백엔드
  |                                    |                                  |
  |  1. 로그인 페이지 접속              |                                  |
  | ---------------------------------> |                                  |
  |                                    |                                  |
  |  2a. [이메일 회원가입] 클릭         |                                  |
  |      이메일, 비밀번호, 닉네임 입력  |                                  |
  | ---------------------------------> |                                  |
  |                                    |  POST /api/auth/register         |
  |                                    | -------------------------------->|
  |                                    |                                  | 검증:
  |                                    |                                  | - 이메일 형식
  |                                    |                                  | - 비밀번호 규칙
  |                                    |                                  | - 이메일 중복
  |                                    |  201 Created                     |
  |                                    |<---------------------------------|
  |                                    |                                  |
  |                                    |  POST /api/auth/login            |
  |                                    | -------------------------------->|
  |                                    |  {access_token, refresh_token}   |
  |                                    |<---------------------------------|
  |                                    |                                  |
  |  3. 대시보드로 이동                 |  토큰 저장 (메모리/httpOnly)     |
  |<---------------------------------- |                                  |
  |                                    |                                  |
  |  --- 또는 ---                       |                                  |
  |                                    |                                  |
  |  2b. [Google로 로그인] 클릭         |                                  |
  | ---------------------------------> |                                  |
  |                                    |  GET /api/auth/google            |
  |                                    | -------------------------------->|
  |                                    |  302 → Google 로그인 페이지       |
  |                                    |<---------------------------------|
  |  Google 로그인 + 권한 승인          |                                  |
  | ---------------------------------> | (Google)                         |
  |                                    |  GET /api/auth/google/callback   |
  |                                    | -------------------------------->|
  |                                    |  302 → /auth/callback?tokens=... |
  |                                    |<---------------------------------|
  |  대시보드로 이동                    |  토큰 추출 및 저장                |
  |<---------------------------------- |                                  |
```

#### 단계 2: 대시보드 (초기 상태)

처음 가입한 사용자는 설정이 없으므로, 대시보드에 설정 가이드가 표시된다.

```
+----------------------------------------------------------+
|  AivaLink Dashboard                       [홍길동] [설정] |
+----------------------------------------------------------+
|                                                          |
|  +----------------------------------------------------+  |
|  |  환영합니다! VTuber를 시작하려면 설정을 완료하세요.   |  |
|  |                                                    |  |
|  |  1. [v] 계정 생성 완료                              |  |
|  |  2. [ ] LLM 설정 (AI 모델 선택)         [설정하기]  |  |
|  |  3. [ ] TTS 설정 (음성 출력 선택)        [설정하기]  |  |
|  |  4. [ ] ASR 설정 (음성 인식 선택)        [설정하기]  |  |
|  |  5. [ ] 캐릭터 생성                      [만들기]   |  |
|  +----------------------------------------------------+  |
|                                                          |
|  내 캐릭터                                               |
|  +----------------------------------------------------+  |
|  |  캐릭터가 없습니다. 위의 가이드를 따라 생성하세요.    |  |
|  +----------------------------------------------------+  |
|                                                          |
+----------------------------------------------------------+
```

## 3. 시나리오 2: LLM 제공자 변경

### 3.1 API 키 직접 입력 방식

```
사용자                              프론트엔드                           백엔드
  |                                    |                                  |
  |  1. 대시보드 → [설정] →            |                                  |
  |     [LLM 설정] 탭                  |                                  |
  | ---------------------------------> |                                  |
  |                                    |  GET /api/llm-configs            |
  |                                    | -------------------------------->|
  |                                    |  {items: [{name:"내 GPT-4o",...}]}|
  |                                    |<---------------------------------|
  |                                    |                                  |
  |  2. [새 LLM 설정 추가] 클릭        |                                  |
  | ---------------------------------> |                                  |
  |                                    |  GET /api/llm-configs/providers  |
  |                                    | -------------------------------->|
  |                                    |                                  |
  |  3. 제공자: "Claude"               |                                  |
  |     모델: "claude-sonnet-4-20250514"         |                                  |
  |     API 키: "sk-ant-xxx..."        |                                  |
  |     [테스트 연결] 클릭              |                                  |
  | ---------------------------------> |                                  |
  |                                    |  POST /api/llm-configs/test      |
  |                                    |  (간단한 프롬프트로 연결 확인)    |
  |                                    | -------------------------------->|
  |                                    |  {success: true, latency_ms: 850}|
  |                                    |<---------------------------------|
  |                                    |                                  |
  |  4. "연결 성공!" → [저장] 클릭     |                                  |
  | ---------------------------------> |                                  |
  |                                    |  POST /api/llm-configs           |
  |                                    | -------------------------------->|
  |                                    |  201 Created                     |
  |                                    |<---------------------------------|
  |                                    |                                  |
  |  5. 캐릭터 설정에서 LLM 변경       |                                  |
  |     아이바 → LLM: "Claude Sonnet"  |                                  |
  | ---------------------------------> |                                  |
  |                                    |  PUT /api/characters/{id}        |
  |                                    |  {llm_config_id: "new-uuid"}     |
  |                                    | -------------------------------->|
  |                                    |  200 OK                          |
  |                                    |<---------------------------------|
  |                                    |                                  |
  |  6. (인스턴스 실행 중이면)          |                                  |
```

### 3.2 OAuth 연결 방식 (Gemini 예시)

```

  |                                    | -------------------------------->|
  |                                    |  302 → Google OAuth (Gemini API  |
  |                                    |         scope 요청)              |
  |                                    |<---------------------------------|
  |                                    |                                  |
  |  3. Google에서 Gemini API 접근     |                                  |
  |     권한 승인                       |                                  |
  | ---------------------------------> | (Google)                         |
  |                                    |                                  |
  |                                    |  콜백 → oauth_token 저장         |
  |                                    | -------------------------------->|
  |                                    |  {oauth_token_id: "token-uuid"} |
  |                                    |<---------------------------------|
  |                                    |                                  |
  |  4. "Google 계정 연결 완료!"       |                                  |
  |     모델: "gemini-2.0-flash"       |                                  |
  |     [저장]                          |                                  |

  |                                    |   oauth_token_id: "token-uuid"}  |
  |                                    | -------------------------------->|
  |                                    |  201 Created                     |
  |                                    |<---------------------------------|
```

## 8. 에러 케이스 및 대응

### 8.1 인증 관련 에러

| 에러 상황 | 사용자 경험 | 대응 |
|-----------|-------------|------|
| Access Token 만료 | API 호출 실패 (401) | 프론트엔드가 자동으로 Refresh Token으로 갱신 시도. 성공 시 원래 요청 재시도. |
| Refresh Token 만료 | 자동 갱신 실패 (401) | 로그인 페이지로 리다이렉트. "세션이 만료되었습니다. 다시 로그인해주세요." |
| Refresh Token 탈취 감지 | 모든 세션 무효화 | "보안 문제가 감지되어 모든 기기에서 로그아웃되었습니다. 비밀번호를 변경해주세요." |
| Google OAuth 실패 | OAuth 콜백 에러 | "Google 로그인에 실패했습니다. 다시 시도해주세요." + 재시도 버튼 |

```
JWT 자동 갱신 흐름:

API 요청 → 401 응답 → refresh_token으로 /api/auth/refresh 호출
                          |
                     성공 (200)                    실패 (401)
                          |                            |
                   새 토큰 저장                   로그인 페이지로
                   원래 요청 재시도                리다이렉트
```

### 8.2 인스턴스 관련 에러

| 에러 상황 | 사용자 경험 | 대응 |
|-----------|-------------|------|
| 인스턴스 시작 실패 | "인스턴스를 시작할 수 없습니다" | 에러 메시지 표시 + 설정 확인 안내 |
| 인스턴스 비정상 종료 | WebSocket 연결 끊김 | "연결이 끊겼습니다. 재연결 중..." → 자동 재연결 시도 (최대 5회, 지수 백오프) |
| 인스턴스 제한 초과 | "동시에 1개의 인스턴스만..." | 기존 인스턴스 중지 후 재시도 안내 |
| LLM API 키 무효 | 인스턴스 시작 시 검증 실패 | "LLM API 키가 유효하지 않습니다. 설정을 확인해주세요." + 설정 페이지 링크 |

### 8.3 파이프라인 관련 에러

| 에러 상황 | 사용자 경험 | 대응 |
|-----------|-------------|------|
| ASR 인식 실패 | "음성을 인식하지 못했습니다" | 채팅 패널에 안내 메시지. 재시도 유도. |
| LLM 응답 실패 | "AI 응답을 생성할 수 없습니다" | API 키 확인 안내. 제공자 서비스 상태 확인. |
| LLM Rate Limit | "요청이 너무 많습니다" | "잠시 후 다시 시도해주세요. (약 30초)" + 카운트다운 |
| TTS 변환 실패 | "음성을 생성할 수 없습니다" | 텍스트 응답은 표시, 음성 없이 진행. TTS 설정 확인 안내. |
| 커스텀 음성 모델 로드 실패 | "음성 모델을 불러올 수 없습니다" | 기본 TTS로 폴백 + 설정 확인 안내 |

```
에러 시 폴백 전략:

ASR 실패 → "음성을 인식하지 못했어요. 다시 말씀해주시거나 텍스트로 입력해주세요."
             (텍스트 입력은 항상 사용 가능)

LLM 실패 → "죄송합니다, 잠시 응답을 생성할 수 없습니다. 잠시 후 다시 시도해주세요."
             (LLM 폴백은 없음 - 핵심 기능이므로 에러 표시)

TTS 실패 → 텍스트 응답은 정상 표시
             Live2D 감정 표현은 정상 작동
             "음성 출력을 사용할 수 없습니다. TTS 설정을 확인해주세요."
```
